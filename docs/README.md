## 🚀 如何使用

### 方式 1: 规则自动应用

所有规则文件的 `alwaysApply: true` 标记表示这些规则会**自动应用**到你的 Java 项目中。

Cursor AI 会在你编写代码时，自动参考这些规则提供建议。

### 方式 2: 主动引用规则

在与 Cursor AI 对话时，可以明确引用特定规则：

```
请根据 @java.mdc 和 @spring-boot.mdc 创建一个用户管理模块
```

### 方式 3: 代码生成时应用

```
根据 Spring Boot 开发规则，帮我创建一个完整的 CRUD Controller
```

Cursor AI 会自动应用 `spring-boot.mdc` 和 `api-design.mdc` 规则。

## 🔧 自定义规则

### 添加新规则

在本目录下创建新的 `.mdc` 文件：

```markdown
---
alwaysApply: true
---

# 你的规则标题

## 规则内容
...
```

### 修改现有规则

直接编辑对应的 `.mdc` 文件即可，Cursor 会自动识别更新。

### 禁用某个规则

将规则文件头部的 `alwaysApply` 改为 `false`：

```markdown
---
alwaysApply: false
---
```

## 📚 与其他文档的关系

### rules/ vs 上层 .mdc 文件

**rules/ 目录** (规则文件):
- 简洁、核心规范
- 自动应用到所有代码生成
- 主要是规则和原则

**上层 .mdc 文件** (知识库文档):
- 详细、完整的教程
- 需要主动 `@` 引用
- 包含完整代码示例和说明

**推荐使用方式**:
1. 规则文件自动生效，保证代码质量
2. 需要详细学习时，引用上层知识库文档
